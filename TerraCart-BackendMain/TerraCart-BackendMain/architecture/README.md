# Multi-Level Menu System - Quick Start Guide

## Overview

This architecture implements a three-tier menu system:
1. **Global Menu** (Super Admin) → Base template for all franchises
2. **Franchise Menu** (Franchise Admin) → Cloned from global, independent
3. **Cart Menu** (Cart Admin) → Cloned from franchise, unique per cart

## Key Features

✅ **Complete Data Isolation** - Each menu level has its own collection  
✅ **No Data Leakage** - Queries are scoped by `franchiseId` and `cartId`  
✅ **Automatic Cloning** - Menus are auto-cloned when franchises/carts are created  
✅ **Unique IDs** - Each menu item gets a new `_id` (no shared references)  
✅ **Role-Based Access** - Proper access control at each level

## Usage Flow

### 1. Create Global Menu (Super Admin)

```javascript
// POST /api/global-menu
{
  "name": "Margherita Pizza",
  "description": "Classic pizza with tomato and mozzarella",
  "category": "Pizza",
  "price": 12.99,
  "taxRate": 5,
  "isAvailable": true
}
```

### 2. Create Franchise (Auto-clones Global Menu)

```javascript
// POST /api/franchises
{
  "name": "Downtown Franchise",
  "franchiseAdminEmail": "admin@downtown.com",
  "address": "123 Main St",
  "phone": "555-0100"
}
```

**What happens:**
- Franchise is created
- Global menu is automatically cloned to franchise menu
- Franchise menu is now independent (changes to global don't affect it)

### 3. Create Cart (Auto-clones Franchise Menu)

```javascript
// POST /api/carts
{
  "name": "Cart 1",
  "franchiseId": "franchise_id_here",
  "cartAdminEmail": "cart1@downtown.com",
  "location": "Lobby"
}
```

**What happens:**
- Cart is created
- Franchise menu is automatically cloned to cart menu
- Cart menu is now unique (not shared with other carts)
- Any old menu data for this cart is deleted first

### 4. Get Menus

```javascript
// Get global menu (Super Admin only)
GET /api/global-menu

// Get franchise menu (Franchise Admin or Super Admin)
GET /api/franchises/:franchiseId/menu

// Get cart menu (Cart Admin, Franchise Admin, or Super Admin)
GET /api/carts/:cartId/menu
```

## Data Isolation Guarantees

### 1. **Scoped Queries**
All menu queries are scoped by `franchiseId` or `cartId`:

```javascript
// Franchise menu query
const items = await FranchiseMenuItem.find({ franchiseId: franchiseId });

// Cart menu query
const items = await CartMenuItem.find({ cartId: cartId });
```

### 2. **Cleanup Before Cloning**
Before cloning, any existing menu data is deleted:

```javascript
// In cloneFranchiseMenuToCart
await CartMenuItem.deleteMany({ cartId: cartId }); // Delete old data
// Then clone fresh menu
```

### 3. **Unique IDs**
Each cloned item gets a new `_id`:

```javascript
const cartItems = franchiseItems.map((item) => ({
  cartId: cartId,
  // ... other fields
  // _id is auto-generated by MongoDB
}));
```

### 4. **Validation in Controllers**
Controllers validate that items belong to the correct cart/franchise:

```javascript
const item = await CartMenuItem.findOneAndUpdate(
  { _id: itemId, cartId: cartId }, // CRITICAL: Ensure item belongs to this cart
  { isAvailable: isAvailable }
);
```

## Preventing Data Leakage

### ✅ What We Do

1. **Separate Collections** - Each menu level has its own collection
2. **Cleanup Before Clone** - Delete old data before cloning
3. **Scoped Queries** - Always filter by `franchiseId` or `cartId`
4. **Validation** - Check item belongs to correct cart/franchise
5. **Unique IDs** - Each item gets new `_id` (no shared references)

### ❌ What We Don't Do

1. **No Shared References** - Items are cloned, not referenced
2. **No Global Queries** - Never query without `franchiseId`/`cartId`
3. **No Cross-Cart Updates** - Updates are scoped to specific cart
4. **No Reusing Old Data** - Always delete before cloning

## Role-Based Access Control

### Super Admin
- ✅ Create/update/delete global menu
- ✅ Create franchises
- ✅ View all franchise menus
- ✅ View all cart menus

### Franchise Admin
- ✅ View/update own franchise menu
- ✅ Create carts in own franchise
- ✅ View cart menus in own franchise
- ❌ Cannot access other franchises

### Cart Admin
- ✅ View own cart menu
- ✅ Toggle item availability in own cart
- ❌ Cannot change prices or structure
- ❌ Cannot access other carts

## Performance Considerations

### Indexes

All models have appropriate indexes:

```javascript
// Franchise menu indexes
franchiseMenuItemSchema.index({ franchiseId: 1, category: 1 });
franchiseMenuItemSchema.index({ franchiseId: 1, isAvailable: 1 });

// Cart menu indexes
cartMenuItemSchema.index({ cartId: 1, category: 1 });
cartMenuItemSchema.index({ cartId: 1, isAvailable: 1 });
cartMenuItemSchema.index({ franchiseId: 1, cartId: 1 });
```

### Query Optimization

- Use `.lean()` for read-only queries
- Use compound indexes for common query patterns
- Scope queries by `franchiseId`/`cartId` first

## Migration from Existing System

If you're migrating from an existing system:

1. **Export existing menu data**
2. **Import to GlobalMenu collection** (Super Admin level)
3. **Create franchises** (will auto-clone global menu)
4. **Create carts** (will auto-clone franchise menu)
5. **Verify data isolation** (run tests)

## Troubleshooting

### Issue: Old menu data appears in new cart

**Solution**: Ensure cleanup happens before cloning:
```javascript
await CartMenuItem.deleteMany({ cartId: cartId }); // Before cloning
```

### Issue: Menu changes affect other carts

**Solution**: Verify queries are scoped:
```javascript
// ❌ Wrong
const items = await CartMenuItem.find({});

// ✅ Correct
const items = await CartMenuItem.find({ cartId: cartId });
```

### Issue: Performance issues with large menus

**Solution**: Add indexes and use lean queries:
```javascript
const items = await CartMenuItem.find({ cartId: cartId }).lean();
```
